# Contact-manage

Мультитенантный Telegram-бот для управления контактами в Telegram-аккаунтах пользователей.

## Что важно понимать

Обычный Telegram Bot API не умеет управлять личными контактами аккаунта. Поэтому в проекте используется гибридная схема:
1. `aiogram` управляет чатом бота и кнопочным интерфейсом.
2. `Telethon` работает от имени подключенного Telegram-аккаунта пользователя через MTProto.
3. Сессия пользователя хранится на сервере в зашифрованном виде.

Локальную телефонную книгу смартфона бот не меняет. Он работает только с Telegram cloud contacts.

## Возможности

Для каждого пользователя доступны:
1. Подключение своего Telegram-аккаунта по номеру телефона.
2. Ввод кода подтверждения, который присылает Telegram.
3. Ввод 2FA-пароля, если он включен.
4. Сохранение Google Sheets источника.
5. Сохранение Yandex CSV источника.
6. Переключение активного источника кнопкой.
7. Импорт следующей пачки контактов.
8. Сброс прогресса импорта.
9. Удаление всех Telegram-контактов в подключенном аккаунте.

## Интерфейс

Интерфейс сделан в формате одного окна:
1. Бот старается редактировать одно и то же сообщение, а не спамить новыми.
2. Основная навигация выполнена через inline-кнопки.
3. Для шагов, где нужны данные от пользователя, бот открывает отдельный экран и просит ввести значение.
4. После обработки введенное пользователем сообщение удаляется.

То есть команды в обычной работе не нужны. После первого открытия бота всё управление идет через кнопки.

## Формат таблицы контактов

Ожидаются 3 колонки в таком порядке:
1. `name`
2. `nickname`
3. `phone`

Первая строка считается заголовком и пропускается.

Поддерживаются строки:
1. имя + телефон
2. имя + username
3. только телефон
4. только username

## Установка

```bash
make install
cp .env.example .env
```

Или вручную:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
```

## Настройка `.env`

На сервере нужно заполнить:
1. `BOT_TOKEN` - токен бота от BotFather
2. `BOT_OWNER_ID` - Telegram user id владельца, либо `0`, если доступ не ограничивается одним пользователем
3. `TG_API_ID` - MTProto API ID от `my.telegram.org`
4. `TG_API_HASH` - MTProto API HASH от `my.telegram.org`
5. `SESSION_SECRET` - ключ шифрования для сохраненных Telegram-сессий
6. `DATABASE_PATH` - путь до SQLite базы
7. `BATCH_SIZE` - желаемый размер пачки
8. `MAX_BATCH_SIZE` - жесткий лимит на размер пачки
9. `SLEEP_BETWEEN_REQUESTS_SEC` - базовая задержка между запросами
10. `REQUEST_JITTER_SEC` - случайная добавка к задержке
11. `IMPORT_COOLDOWN_SEC` - пауза между пачками импорта
12. `DELETE_COOLDOWN_SEC` - пауза между полными удалениями контактов
13. `LOGIN_CODE_COOLDOWN_SEC` - пауза между запросами кода входа

Сгенерировать `SESSION_SECRET` можно так:

```bash
python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
```

## Настройка Google Sheets

Сервисный аккаунт Google **не нужен**. Бот читает таблицу напрямую через публичный CSV-экспорт.

1. Откройте Google Sheet → «Настройки доступа» → «Все, у кого есть ссылка» → **Читатель**.
2. В боте нажмите кнопку «Источник Google».
3. Отправьте полную ссылку на таблицу, например:
   `https://docs.google.com/spreadsheets/d/1Abc.../edit?usp=sharing`
4. При необходимости на второй строке укажите имя листа (по умолчанию `Sheet1`).

Первая строка таблицы должна быть заголовком: `name, nickname, phone`.

## Настройка Yandex

1. Получите публичную CSV-ссылку из Yandex Tables.
2. В боте нажмите `Set Yandex CSV`.
3. Отправьте ссылку.

## Управление ботом на сервере

Доступны команды:

```bash
make install
make start
make stop
make restart
make status
make logs
```

Что делает каждая команда:
1. `make install` - создает `.venv` и устанавливает зависимости
2. `make start` - запускает бота в фоне
3. `make stop` - останавливает бота
4. `make restart` - перезапускает бота
5. `make status` - показывает, работает ли бот
6. `make logs` - показывает лог бота

Если обязательные переменные в `.env` не заполнены, `make start` завершится с понятной ошибкой еще до запуска.

## Как пользоваться ботом

1. Откройте чат с ботом и нажмите стандартную кнопку `Start` в Telegram.
2. В окне бота нажмите `Connect Telegram account`.
3. Отправьте номер телефона в международном формате.
4. Отправьте код, который пришлет Telegram.
5. Если включен 2FA, отправьте пароль.
6. Настройте источник через `Set Google Sheet` или `Set Yandex CSV`.
7. Выберите активный источник кнопкой `Use Google` или `Use Yandex`.
8. Нажмите `Import next 200`.

## Ограничения и защита аккаунта

Чтобы уменьшить риск ограничений со стороны Telegram, в проекте включены:
1. работа только в private chat для чувствительных шагов;
2. пауза между запросами импорта;
3. случайный jitter, чтобы запросы не были слишком равномерными;
4. cooldown между пачками импорта;
5. cooldown между полными удалениями контактов;
6. cooldown между повторными запросами кода входа;
7. при `FloodWait` бот ждет нужное время и продолжает работу, пока текущая пачка не завершится.

Важно:
1. Даже с этими мерами Telegram не дает гарантии от ограничений.
2. Слишком агрессивные объемы и слишком частые импорты всё равно могут привести к ограничениям.
3. Если для аккаунта Telegram вернул `FloodWait`, бот теперь не обрывает задачу, а ждет и продолжает.

## Безопасность

1. Пользователи не видят `TG_API_ID` и `TG_API_HASH`, они остаются на сервере.
2. Telegram-сессии хранятся в SQLite в зашифрованном виде через `SESSION_SECRET`.
3. Сообщения пользователя с номером телефона, кодом, паролем и ссылками после обработки удаляются.
4. Если токен бота когда-либо засветился в репозитории или переписке, его нужно перевыпустить через BotFather.
